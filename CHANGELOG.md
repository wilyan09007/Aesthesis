# Changelog

All notable changes to Aesthesis are recorded here. Format inspired by Keep a Changelog; versions follow the project's 4-digit `MAJOR.MINOR.PATCH.MICRO` scheme.

## [0.3.1.0] - 2026-05-02

### Added

- **Agent-paste prompts per insight — close the dev loop.** Each insight now ships a Markdown prompt the user copies to clipboard and pastes straight into Claude Code, Cursor, Aider, or Copilot Chat. The prompt names the specific UI element that caused the brain response (label + visible text + location + visual anchors), states the change, and lists 2-4 falsifiable acceptance criteria. The prompt is the deliverable — the brain told us *what* changed and *why*; the prompt is *how* to fix it. Full design log in `ASSUMPTIONS_AGENT_PROMPT.md` (§§1-22).
- **Vision-grounded UI element pinpointing.** Gemini now emits a structured `target_element` per event with three converging anchors (visible text, location hint, visual anchors) plus a normalized bounding box. The brain signal narrows the search — friction_anxiety constrains the target to dense / confusing controls, motor_readiness biases toward CTAs, etc. Failure modes covered: out-of-range bbox coercion (0..1 vs 0..1000), unclear-label escape hatch when a single element can't be committed, fuzzy `change_type` enum coercion. See `aesthesis_app/aesthesis/annotate.py` and the rewritten `prompts.py` `INSIGHT_PROMPT_TEMPLATE`.
- **Confidence-calibrated prompt branches.** Three prompt templates render deterministically based on Gemini's emitted confidence: `standard` (≥0.7), `cautious` (0.4-0.7, adds a "verify before applying" line), `unclear` (<0.4 or unclear-label, asks the agent to investigate rather than commit). Renderer at `aesthesis_app/aesthesis/prompt_renderer.py`. The "permission to deviate" clause keeps agents from over-anchoring on the literal `desired_state` wording.
- **Annotated screenshots inline in the response.** PIL bbox overlay (Aesthesis accent stroke + soft glow halo) is base64-embedded on each insight as `annotated_screenshot_b64`. The InsightCard renders it as a thumbnail with click-to-lightbox; users can also save the JPEG via a secondary action. Sidesteps Modal's scale-to-zero ephemeral filesystem (cleanup_uploads deletes the run dir after each request) without needing a Modal volume or Vercel Blob plumbing.
- **Screenshot resolution bump 800 → 1600px.** ffmpeg now scales to `min(1600, native_width)` instead of forced 800. Captures the readable detail Gemini needs on dense UIs (small icons, body-copy hierarchy, table rows) without paying the full 1920×1080 token-tile tax. ~3× the per-image Gemini token cost per event, still negligible absolute on Flash. Three call sites in `screenshots.py` updated.
- **Goal-as-prior strengthening in the Gemini prompt.** When the user states a goal ("evaluate the pricing flow"), the prompt biases element selection toward elements directly related to that goal unless the brain signal points elsewhere. Free prompt-copy improvement; no code change.
- **Verbose structured logging across the synthesizer.** Every step boundary (`synth.insights`, `synth.parse`, `synth.coerce`, `synth.enrich`, `synth.render`, `annotate.coerce`, `annotate`) emits an INFO/DEBUG line with stable `extra` fields (`run_id`, `insight_idx`, counts, elapsed_ms). Per-event degradations are searchable without grepping per-event noise.
- **`InsightCard.tsx` rewrite.** Expanded view now shows target-element block (with annotated screenshot thumb), proposed-change block (from / to / why with change-type pill), acceptance-criteria checklist, brain-signal pills, and a primary "Copy prompt for AI agent" button with copy-success toast. Lightbox for full-size annotated screenshot. Backboard "personalized fix" demoted to a secondary "Compare to past runs" action — Backboard is the right tool for trend chat, not prompt synthesis. Card border tints amber on the unclear-confidence branch so users read the prompt with calibrated trust before pasting.
- **Real tests, no mocks** (`tests/test_app_annotate.py`, `tests/test_app_prompt_renderer.py`): 34 new tests covering the bbox coercion table (0..1 passthrough, 0..1000 rescale, tiny-overshoot clamp, unsalvageable drop, degenerate / inverted / wrong-length / non-numeric inputs), the PIL overlay round-trip (real Pillow, JPEG magic-byte verification), all three prompt-renderer branches (standard / cautious / unclear), determinism (same input → identical output), missing-visible-text fallback, marketing-vocabulary lint, and the "permission to deviate" clause assertion. Pillow is required in the dev env — tests refuse to skip if it's missing, per the project's `feedback_no_mocks` posture.
- **`ASSUMPTIONS_AGENT_PROMPT.md`** — comprehensive research and design log. §1-§20 cover the v1 spec; §21 the deep accuracy research (5-failure-mode taxonomy, EasyOCR pre-pass, Set-of-Marks numbered overlays, multi-frame context for change-driven events, two-pass crop-and-confirm verifier, cross-event consistency boost, considered-and-rejected techniques); §22 is the implementation log capturing what was actually shipped, what was deferred to v1.1, what was researched mid-implementation, and the risks consciously accepted.

### Changed

- **`Insight` schema extended with optional fields** (mirror in `aesthesis-app/lib/types.ts`): `target_element` (`TargetElement | null`), `proposed_change` (`ProposedChange | null`), `acceptance_criteria` (`string[]`), `confidence` (`number`), `agent_prompt` (`string`), `annotated_screenshot_b64` (`string | null`). The legacy `recommendation` string stays as a derived back-compat field so existing chart hover tooltips keep working without a frontend rewrite.
- **`Pillow>=10.0` added** to `requirements-app.txt` and the Modal orchestrator image — drives the bbox overlay. Soft fallback in `annotate.py` if PIL ever goes missing (returns null, prompt still renders without the image).

## [0.3.0.0] - 2026-04-25

### Added

- **Phase 2 capture pipeline (URL → BrowserUse → live frames + MP4 → existing analyzer).** Paste a URL on the Capture screen, BrowserUse drives a real Chromium tab against the goal, the frontend watches the agent work in real time, and the captured 30-second MP4 flows straight into the existing TRIBE + Gemini pipeline. End-to-end no-mocks. DESIGN.md §§4.1, 4.2, 4.2b implemented per the locked plan at `~/.claude/plans/dazzling-tinkering-sedgewick.md` (decisions D11-D33).
- **`aesthesis_app/aesthesis/capture/` package** — `protocol.py` (Pydantic wire types: `RunRequest`, `AuthSpec`, `WSCaptureComplete`, `WSCaptureFailed`, `CachedDemoEntry`), `streamer.py` (5-tier adaptive CDP screencast streamer with hysteresis-gated tier walks and a hard 2 fps floor per D9), `runner.py` (parent-side lifecycle: subprocess management, 90s wall-clock SIGKILL chain per D1, cross-platform Chromium zombie sweep by both PID and name per D26, 3-second WS-empty grace window per D27, last-lifecycle replay on reconnect per D32, single-tenant cap=1 per D19).
- **`aesthesis_app/aesthesis/browser_agent.py`** — the `python -m aesthesis.browser_agent` subprocess entry. Launches Playwright Chromium with `--remote-debugging-port`, opens a CDP session for `Page.startScreencast`, hands the same Chromium to BrowserUse via `BrowserSession(cdp_url=...)` (the only coexistence pattern that lets us record AND drive the same tab — see `ASSUMPTIONS_PHASE2_CAPTURE.md` A2). CDP frames serve double duty: streamed live to stdout AND stitched into an H.264 MP4 via ffmpeg `image2pipe`. Action history written to `actions.jsonl`. Loud failure on any setup or runtime error with typed reasons (`timeout`, `crashed`, `navigation_error`, `setup_error`).
- **Five new endpoints in `main.py`:** `POST /api/run` (start a capture; 409 cap=1 per D19), `WS /api/stream/{run_id}` (binary JPEG frames + JSON lifecycle messages per D30c, with replay on reconnect per D32), `GET /api/run/{run_id}/video` (download captured MP4), `POST /api/analyze/by-run/{run_id}` (analyze captured MP4 by reference, retains artifacts on failure per D33), `GET /api/cached-demos` + `GET /api/cached-demos/{filename}` (D29 stage-day fallback).
- **D15 action-log integration.** `orchestrator._attach_per_event_context` (renamed from `_attach_screenshots`) optionally loads `actions.jsonl` produced by the capture subprocess and stamps `Event.agent_action_at_t` for events whose `timestamp_s` falls within ±0.5s of a recorded action. Skip-path callers pass `action_log_path=None` and behaviour is identical to v1 (regression-tested).
- **Frontend Phase 2 wire-up.** `lib/api.ts` gains `WS_BASE_URL` + `wsUrl()` helper (D18 fixes the relative-WS-URL bug in `LiveStreamPanel`), `startCaptureRun()`, `analyzeByRunId()`, `fetchCapturedVideo()`, `fetchCachedDemos()`. `CaptureView.tsx` is wired to the real backend with 409 / network / capture_failed error UI and a "Use cached demo" button on failure when the URL matches a manifest entry. `app/page.tsx` skips `AssessView` on the capture path and auto-fires `analyzeByRunId` after a 3-second confirm gate (D11).
- **D30 frontend smoothness pipeline.** `LiveStreamPanel.tsx` rewritten as a `<canvas>` + `createImageBitmap` (off-main-thread JPEG decode) + `requestAnimationFrame` paint loop with a 3-frame jitter buffer using a drop-stale policy. Binary WebSocket frames (D30c) eliminate the base64 inflation hop. Decouples render cadence from arrival jitter — fixed-cadence paint reads as smooth even when source frames are bursty.
- **D11 confirm countdown gate.** `AnalyzingView.tsx` shows a 3-second "Captured — Analyzing in 3..." countdown with a Cancel button before firing the analyze call on the capture path. Cheap insurance against burning 55-75s of TRIBE+Gemini compute on garbage captures (login walls, cookie modals).
- **No-mocks Phase 2 test suite** (`aesthesis_app/tests/`): `test_browser_agent_kill.py` is the R4 mandatory regression — spawns a real subprocess against the RFC-5737 unreachable IP `192.0.2.1`, verifies the wall-clock SIGKILL fires, no Chromium zombies survive, and a second run launches cleanly. `test_streamer.py` exercises the 5-tier ladder constants + rolling-FPS math + a real-Chromium integration test that captures actual frames. `test_orchestrator_actions.py` covers D15 action-stamping with real on-disk JSONL files. `test_capture_endpoints.py` mixes fast endpoint tests (404, path-traversal guard, manifest parsing) with a real-subprocess 409 cap test. Every test fails LOUDLY when any environmental requirement is missing — no `skipif`, no fakes, per project memory `feedback_no_mocks` and the explicit `/ship` instruction.
- **`ASSUMPTIONS_PHASE2_CAPTURE.md`** — full research log for the build. 18 documented assumptions covering BrowserUse 0.12 API stability, Playwright recordVideo + CDP coexistence (the load-bearing finding), H.264 codec enforcement, ffmpeg discovery order, CDP screencast frame-pacing semantics, BrowserUse history shape variability, RFC-5737 in tests, psutil cross-platform process walk, the binary WS / JSON control split, langchain SecretStr workaround for Gemini auth, cookie injection ordering, and the test-environment fail-loud policy.
- **Bootstrap upgrades.** `dev.sh` auto-installs `playwright install chromium` on first run if the Chromium binary isn't present (~170-450MB one-time download per OS profile). Probes for system ffmpeg with a graceful fallback note when `imageio-ffmpeg` is bundled. `pyproject.toml` pins `browser-use==0.12.6`, `playwright>=1.49,<2.0`, `psutil>=5.9`, `langchain-google-genai>=4.0`, `imageio-ffmpeg>=0.5`. Python requirement bumped to `>=3.11` (browser-use hard requirement). `requirements-dev.txt` adds `psutil`, `websockets`, `Pillow`.

- **Real cortical brain visualization.** The Results page renders an actual fsaverage5 inflated cortical mesh (both hemispheres), colored per-parcel from TRIBE v2 z-scored activations, with drag-to-rotate. Replaces the prior icosahedron placeholder. Powered by `@react-three/fiber` (no new dependencies). Architecture mirrors Meta's TRIBE v2 demo (three.js + GLBs + per-vertex/per-face attributes) — see `ASSUMPTIONS_BRAIN.md` and `aesthesis-app/UIUX.md` §7.
- **`aesthesis-app/components/BrainCortical.tsx`** — loads two GLBs (left + right hemisphere) via drei `useGLTF`, reads custom `_PARCELID` and `_SULC` vertex attributes baked at build time, drives a per-vertex color buffer (`THREE.DynamicDrawUsage`) on every TR change, mixes in sulcal shading. Falls back to the placeholder `Brain3D` when `parcel_series` is null so deploys can stage independently.
- **`aesthesis-app/lib/colormap.ts`** — diverging blue↔red colormap (RdBu_r) anchored at z=0, with a smoothstep-based sulcal shading mix. Sign-aware: positive z reads warm, negative reads cool, zero is muted neutral.
- **`tribe_service/scripts/bake_parcel_map.py`** — projects the Schaefer-400 atlas onto fsaverage5 via `nilearn.surface.vol_to_surf`, masks cross-hemisphere assignments by parsing label names, writes `data/schaefer400_parcels.npy`. Runs once, output ships in the Modal data volume.
- **`tribe_service/scripts/bake_brain_glbs.py`** — bakes 4 GLBs (left/right × inflated/pial) under `aesthesis-app/public/brain/` with custom `_PARCELID` (uint16) and `_SULC` (float32) per-vertex attributes plus pre-computed normals. Hand-built buffer/bufferView/accessor graph via `pygltflib` to preserve the underscore-prefixed custom attribs through `THREE.GLTFLoader`.
- **`tribe_service/tribe_neural/steps/step2b_parcels.py`** — per-parcel reduction (z-scored across time) running in parallel to the existing 8-ROI pipeline. Loud failures on NaN/Inf inputs, shape mismatches, and unassigned parcels.
- **`TimelineSummary.parcel_series`** — new wire field on the `/api/analyze` response, shape `(n_TRs, 400)` float, optional. Backend emits it when the parcel map is loaded; frontend uses it to color the cortical mesh. ~32 KB per 30s clip.
- **`tests/test_step2b_parcels.py`** — real-data parcel-reduction tests (no mocks). Asserts the shape contract, z-score properties, empty-parcel handling, and loud failure on NaN/Inf inputs. The artifact-dependent test fails with a runbook command when the Schaefer parcel map hasn't been baked.
- **`ASSUMPTIONS_BRAIN.md`** — research log + architectural decisions for the cortical brain implementation. Covers fsaverage5 vertex order, Schaefer projection caveats, GLB custom-attribute survival through GLTFLoader, three.js BufferAttribute update patterns, and the no-mocks testing strategy.
- **`aesthesis-app/UIUX.md`** — comprehensive UI/UX implementation plan with the brain-visualization phase plan (§7), cross-referencing the Meta TRIBE v2 demo's confirmed stack (hand-rolled three.js, two-tier mesh GLBs, per-face shader, pre-baked color streams).

### Changed

- **Insight timestamps clamped to video bounds.** `aesthesis_app/aesthesis/synthesizer.py` now passes `duration_s` into both Gemini prompts and snaps every returned `timestamp_range_s` into `[0, duration_s]`. Insights whose start lies past the video end are dropped with a logged warning. Fixes the bug where insight cards pointed past the end of the video player.
- **Video panel sized to `50vw × 50vh`.** `aesthesis-app/components/VideoPlayer.tsx` resolves the duelling `flex-1` + `aspect-video` feedback loop that made the frame fill the viewport. Inner `flex-1` removed; outer panel given an explicit `height: 50vh`.
- **Brain panel paired with the video.** `aesthesis-app/components/ResultsView.tsx` wraps the brain in its own `50vh × 50vh` panel with header (`Neural state · t = X.Xs`) and a "drag to rotate" caption. Faint pulsing-orb fallback during lazy-load instead of a generic spinner.
- **Brain colormap sign-aware.** `aesthesis-app/components/Brain3D.tsx` squashes z-scores through `tanh(z * 0.7)` and applies asymmetric warm/cool shifts so negative activations read distinctly from "no signal." OrbitControls rotation enabled (zoom/pan still off).
- **`tribe_service/tribe_neural/init_resources.py`** loads the Schaefer parcel map at worker boot when present; logs loudly when absent so the cortical-brain rendering's degradation path is obvious in startup logs.

### Fixed

- **`dev.sh` CRLF safety.** `.env` files with Windows line endings no longer leak `\r` into env var values (which broke `TRIBE_SERVICE_URL` curl checks). Strips CR via `tr -d '\r' < .env` process substitution before sourcing.
- **`dev.cmd`** — Windows entry point. Locates Git for Windows' `bash.exe` and hands `dev.sh` off to it. Skips WSL bash because `dev.sh` expects Windows-side `python` / `npm` / `taskkill.exe`.

## [0.2.0.0] - 2026-04-25

### Added

- **Frontend ↔ backend wired up end to end.** Selecting two MP4s in `aesthesis-app/` now POSTs them to the FastAPI backend at `/api/analyze`, watches the (real) 12–25s pipeline, and renders the live brain timeline + insights + verdict. The `goResults` shortcut and `lib/mockData.ts` are gone.
- **`aesthesis-app/lib/api.ts`** — typed client for `/api/analyze`. `AbortController` cancellation, run-id propagation via `X-Aesthesis-Run-Id` response header, server-timing via `X-Aesthesis-Elapsed-Ms`, structured `console.info` logging keyed on the truncated run id, typed `AnalyzeError(message, status, body, runId)` that unwraps FastAPI `ValidationFailure` detail bodies for human-readable messages.
- **`aesthesis-app/lib/adapt.ts`** — backend `TimelineSummary { roi_series, tr_duration_s }` → `Frame[]` for the recharts/Brain3D consumers, plus `Verdict.summary_paragraph` → `verdict.summary` for the panel. One adapter call on response; components stay backend-shape-agnostic.
- **CORS on the backend.** `CORSMiddleware` reads origins from `CORS_ALLOW_ORIGINS` (defaults cover the Next.js dev server). Preflight + actual POST now succeed cross-origin. The two `X-Aesthesis-*` headers are listed under `expose_headers` so JS can read them.
- **HTTP request middleware** on the backend logs every request's method/path/status/elapsed_ms boundary.
- **Network-aware progress UX in `AnalyzingView`.** The synthetic 4-stage progress bar now stalls at 95% until the response arrives, surfaces a backend error panel with run-id + Retry/Start-Over when the pipeline fails, and labels the last stage as "waiting on Gemini…" to match what's actually happening.
- **`ASSUMPTIONS.md` §17** — full research log for the wire-up: Next.js 16 / React 19 verification (`node_modules/next/dist/docs/`), the Pydantic↔TS type alignment, the network-call + retry strategy, CORS choices, and what's still unverified end-to-end.

### Changed

- **`aesthesis-app/lib/types.ts`** is now a 1:1 mirror of `aesthesis_app/aesthesis/schemas.py`. Backend Pydantic is the single source of truth — divergence fails loudly at build time.
- **`/api/analyze` response carries headers** `X-Aesthesis-Run-Id` and `X-Aesthesis-Elapsed-Ms` so cross-tier debugging is `grep run_id` across browser + uvicorn logs.

### Removed

- **`aesthesis-app/lib/mockData.ts`** — per the project's no-mocks rule, fixture data designed to ship to production has no place in `lib/`. The frontend now only renders real backend output.

## [0.1.0.0] - 2026-04-25

### Added

- **Step 2 (Assess) backend pipeline end-to-end**: takes two MP4 screen recordings and returns the full results-page JSON (per-version brain timelines, insights, head-to-head verdict). Built per `DESIGN.md` §4.4 / §4.5 / §4.7 / §5.15 / §5.16.
- **TRIBE v2 service** (`tribe_service/`): FastAPI + ARQ + Redis architecture per §5.2. Exposes `/process_video_timeline`, `/enqueue_video_timeline`, `/job/{id}`, `/health`. Includes Modal deploy stub (`modal_app.py`).
- **TRIBE pipeline core**: `extract_all` against `NETWORK_KEYS_UX` (8 ROIs), per-TR composites (8), window composites (6), connectivity over `PAIRS_UX` (7 pairs), timeline builder with sliding-window overlay (`window_trs=4` default; `window_trs=6` for std-based composites per R2).
- **TRIBE runner** (`tribe_runner.py`): dispatches to `tribev2.demo_utils.TribeModel.predict`. Heavy deps (torch, V-JEPA 2, DINOv2, Wav2Vec-BERT, LLaMA 3.2) are loaded lazily on first call.
- **Aesthesis app backend** (`aesthesis_app/`): FastAPI `/api/analyze` endpoint accepting two MP4 multipart uploads + optional `goal`. Validates via `ffmpeg.probe` (codec, duration ≤30s, ≤1080p, ≤50MB). Forwards to TRIBE serially per D6, then runs the Insight Synthesizer.
- **Insight Synthesizer**: deterministic event extraction (spike, dominant_shift, sustained, co_movement, trough, flow, bounce_risk) per §4.5 step 1; Gemini 2.0 Flash insight call per version with the prompt + JSON schema verbatim from §4.5; Gemini verdict call with cross-version aggregate metrics. Failures (missing key, malformed JSON, schema mismatch) raise `SynthesizerError` and surface as a 500.
- **Output builder**: assembles the final results-page JSON contract that the (future) Next.js UI consumes.
- **Verbose structured logging** throughout both services using stdlib `logging` with `extra={...}` fields. `LOG_LEVEL=DEBUG` surfaces every step boundary, every shape check, every external call, and every timing measurement.
- **Tests**: pure-function unit tests for all 8 per-TR composites, all 6 window composites, `extract_all`, event extraction, and upload validation. Tests that exercise the full pipeline require a deployed TRIBE GPU service and a real `GEMINI_API_KEY`.
- **`ASSUMPTIONS.md`**: full research log — every TRIBE v2 API detail confirmed, every gap I had to fill, and every decision I made when DESIGN.md was silent or contradictory.
- **`README.md`**: build/run/test/deploy instructions for both services.
